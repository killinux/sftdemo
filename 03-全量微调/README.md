# 全量微调：从零跑通 SFT

## 一、什么是全量微调

全量微调 = 把模型所有参数都拿来训练。本目录包含一个完整的最小 demo：用 5 条数据微调 Qwen2.5-0.5B。

```
全量微调: 5 亿个参数全部参与训练
LoRA:    冻住 5 亿参数，只训练额外加的几百万参数（见 04-LoRA微调/）
```

全量微调效果上限最高，但显存占用大、容易覆盖预训练能力（灾难性遗忘）。实际生产中更推荐 LoRA，但理解全量微调是基础。

## 三、训练过程详解

### 数据格式

每条数据 3 个字段（见 `../02-数据工程/data.json`）：

| 字段 | 含义 | 示例 |
|------|------|------|
| `instruct` | 任务指令 | `请你给敖丙写一首诗：` |
| `input` | 上文/提示 | `碧海生龙子，云中舞雪霜。` |
| `label` | 期望输出 | `恩仇难两忘，何处是家乡？` |

### Label Masking（核心操作）

SFT 和预训练的唯一区别：**不对指令部分计算 loss，只对回答部分计算 loss**。

```
输入序列:  请你给敖丙写一首诗：碧海生龙子，云中舞雪霜。恩仇难两忘，何处是家乡？<eos>
          |_________ 指令+上文 ___________||________ 回答 ________|
labels:   -100  -100  -100 ... -100  -100   恩仇难两忘...    <eos>
          |______ 不算loss，跳过 ______|   |____ 只算这部分 ____|
```

代码实现（`train.py` 关键部分）：

```python
# 拼接完整文本并 tokenize
text = item["instruct"] + item["input"] + item["label"] + tokenizer.eos_token

# 单独 tokenize 指令部分，获取长度
instruct = item["instruct"] + item["input"]
instruct_len = tokenizer(instruct, ...)["input_ids"].size(-1)

# 将指令部分的 label 设为 -100
label[:, :instruct_len] = -100
```

### Epoch（训练轮次）

所有数据从头到尾过一遍 = 1 个 Epoch：

```
Epoch 1:  loss = 4.799    第一遍，完全不会
Epoch 3:  loss = 0.142    第三遍，有点印象了
Epoch 10: loss = 0.0003   第十遍，滚瓜烂熟
```

本 demo：5 条数据，batch size=2，每个 epoch 3 步，10 个 epoch 共 30 步。

### 训练参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 基座模型 | Qwen/Qwen2.5-0.5B | 5 亿参数 |
| num_train_epochs | 10 | 训练轮数 |
| per_device_train_batch_size | 2 | 每批 2 条 |
| logging_strategy | epoch | 每轮打印一次 |

### 参数变化

训练本质是微调 5 亿个参数的数值，每个参数平均只变了 0.000066：

```
训练前: 参数 = [0.0234, -0.1567, 0.8912, ...]
训练后: 参数 = [0.0234, -0.1566, 0.8913, ...]
                         ↑ 变了一点点  ↑ 变了一点点
```

用 `compare.py` 可以看到具体变化量。用 `show_layers.py` 可以看模型结构和每层参数量。

## 四、推理验证

### 测试 1：给指令+上文（训练时见过的完整输入）

```
输入: 请你给敖丙写一首诗：碧海生龙子，云中舞雪霜。
预测: 恩仇难两忘，何处是家乡？   ✅ 和标签完全一致
```

5 条全部精确命中。

### 测试 2：只给指令（训练时没见过的输入方式）

```
输入: 请你给敖丙写一首诗：
预测: （空）                     ❌ 什么都没生成
```

部分能答对，部分答不上来。这不是 bug——**模型只学到了你让它学的东西**。

## 五、运行方式

```bash
# 安装依赖
pip install torch transformers accelerate

# 查看模型结构
python show_layers.py

# 训练（输出保存到 ./sft_output）
python train.py

# 推理验证
python infer.py

# 查看参数变化
python compare.py
```

基座模型约 1GB，首次运行自动从 HuggingFace 下载。

## 六、全量微调的局限

| 问题 | 说明 |
|------|------|
| 显存占用大 | 7B 模型全量微调需要 60GB+ 显存 |
| 灾难性遗忘 | 调所有参数，容易把预训练学到的能力改坏 |
| 存储成本高 | 每个任务保存一份完整模型 |
| 容易过拟合 | 数据量小时尤其严重（本 demo 就是典型） |

解决方案 → 见 `04-LoRA微调/`
