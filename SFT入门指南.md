# SFT 入门指南：让大模型学会听话

## 一、什么是大模型

大模型就是**一大堆数字**（叫"参数"），比如 Qwen2.5-0.5B 有 5 亿个参数。

这些数字组成一个流水线：

```
你输入一句话 → 模型一个字一个字地预测下一个字 → 输出回答
```

模型内部长这样：

```
"请你给哪吒写一首诗"
      ↓
  Embedding 层（1.36亿参数）     ← 查字典，把每个字变成一串数字
      ↓
  Transformer 第 1 层           ← 浅层：认字、识别词性
  Transformer 第 2 层
  ...
  Transformer 第 12 层          ← 中层：理解句子含义
  ...
  Transformer 第 24 层          ← 深层：综合理解，构思答案
      ↓
  输出层                        ← 预测下一个字
```

- **Embedding** = 认字（查表，把文字变成数字向量，每个字互不影响）
- **Transformer** = 读懂句子（让字和字之间互相看，理解上下文）
- 24 层结构一样（注意力 + 全连接），但各自学到不同深度的理解

这类模型只用 **Decoder**（生成型），每个字只能看到前面的字，不能偷看后面的，因为后面的字还没生出来。

## 二、什么是 SFT

SFT = Supervised Fine-Tuning = 有监督微调。

大模型的训练分两个阶段：

```
预训练：读了几万亿字的书（维基百科、网页、代码...）
        → 学会了语言能力，但不听指令，像个读了很多书但没上过班的人

SFT：   用标注好的"问题→回答"数据再训练一下
        → 学会了按指令回答问题，像入职第一天老板教你怎么干活
```

**一句话：预训练给了模型语言能力，SFT 教它听话。**

## 三、SFT 的核心原理

SFT 和预训练的代码流程基本一致，唯一的区别：**不对用户的指令部分计算 loss，只对回答部分计算 loss**。

### loss 是什么

模型预测下一个字时会给出概率：

```
正确答案是 "恩"

训练前: "恩" 的概率 5%   → loss 大（猜错了）
训练后: "恩" 的概率 95%  → loss 小（猜对了）
```

**loss 大 = 猜错了，loss 小 = 猜对了。训练就是不断调参数让 loss 变小。**

### 怎么做到"只对回答算 loss"

把指令部分的 label 设为 -100，PyTorch 会自动跳过：

```
输入序列:  请你给敖丙写一首诗：碧海生龙子，云中舞雪霜。恩仇难两忘，何处是家乡？<eos>
          |_________ 指令+上文 ___________||________ 回答 ________|
labels:   -100  -100  -100 ... -100  -100   恩仇难两忘...    <eos>
          |______ 不算loss，跳过 ______|   |____ 只算这部分 ____|
```

类比：老师出题 "1+1=?"，SFT 只让学生记住答案 "2"，不用学背题目。

## 四、训练过程

### 数据

本 demo 用了 5 条哪吒角色诗歌数据：

| instruct（指令） | input（上文） | label（期望输出） |
|------|------|------|
| 请你给哪吒写一首诗： | 哪吒降世，意气飞扬... | 红绫缠腕，风火踏浪... |
| 请你给敖丙写一首诗： | 碧海生龙子，云中舞雪霜。 | 恩仇难两忘，何处是家乡？ |
| ...共 5 条 | | |

### Epoch（轮次）

把所有数据从头到尾过一遍 = 1 个 Epoch，就像刷题：

```
Epoch 1:  loss = 4.799    第一遍，完全不会
Epoch 3:  loss = 0.142    第三遍，有点印象了
Epoch 10: loss = 0.0003   第十遍，滚瓜烂熟
```

本 demo 共 5 条数据，batch size 为 2，每个 epoch 3 步，10 个 epoch 共 30 步，38 秒训完。

### 模型发生了什么变化

训练的本质是微调 5 亿个参数的数值，每个参数平均只变了 0.000066：

```
训练前: 参数 = [0.0234, -0.1567, 0.8912, ...]
训练后: 参数 = [0.0234, -0.1566, 0.8913, ...]
                         ↑ 变了一点点  ↑ 变了一点点
```

5 亿个参数各变一点点，累积起来就让模型学会了"怎么回答这 5 首诗"。这种规律无法用人类语言直接表示，它分散编码在整个网络里。

## 五、推理验证

### 测试 1：给指令+上文（训练时见过的完整输入）

```
输入: 请你给敖丙写一首诗：碧海生龙子，云中舞雪霜。
预测: 恩仇难两忘，何处是家乡？   ✅ 和标签完全一致
```

5 条全部精确命中。

### 测试 2：只给指令（训练时没见过的输入方式）

```
输入: 请你给敖丙写一首诗：
预测: （空）                     ❌ 什么都没生成
```

部分能答对，部分答不上来。

### 为什么会这样

模型训练时见到的永远是 `指令+上文 → 回答`，从没见过 `指令 → 回答`。就像服务员只培训了 3 道菜的话术，顾客问第 4 道菜就懵了。

这不是 bug，恰恰证明了 **模型只学到了你让它学的东西**。

## 六、关键概念速查

| 概念 | 一句话解释 |
|------|-----------|
| 预训练 | 读几万亿字的书，获得语言能力 |
| SFT | 用标注数据微调，教模型听指令 |
| Embedding | 查字典，把文字变成数字向量 |
| Transformer | 让字和字互相看，理解上下文 |
| Decoder | 只能看前文不能看后文的 Transformer，用于生成 |
| Encoder | 能看全文的 Transformer，用于理解 |
| Loss | 模型预测和正确答案的差距 |
| Epoch | 所有训练数据过一遍 |
| Label = -100 | 告诉模型这个位置不用学（SFT 的核心操作） |
| 参数 | 模型里的数字，训练就是调这些数字 |

## 七、运行方式
