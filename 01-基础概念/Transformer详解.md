# Transformer 详解：从原理到运行结果

配合 `transformer_demo.py` 一起看。demo 用纯 PyTorch 从零搭建了一个 Transformer 语言模型（0.21M 参数），训练它学会几句中文的续写。

## 整体流程

```
训练语料（5句中文） → 构建词表(54字) → 搭建模型 → 训练100轮 → 学会续写
```

---

## 一、准备数据

### 1.1 构建词表

扫描语料，找出所有不重复的字符，每个字给一个整数编号：

```
"天"→12, "问"→49, "一"→2, "号"→10, "成"→22, "功"→7
```

这就是最简单的 Tokenizer。大模型也是这个过程，只不过：
- 我们的 demo：54 个字符，一个字一个 token
- Qwen2.5：151,936 个子词（subword），用 BPE 算法切分

### 1.2 切分训练样本

用长度 32 的滑动窗口，每次往后移一个字，切出 101 个样本。

每个样本的任务都是 **"根据前面的字，预测下一个字"**：

```
输入: [天, 问, 一, 号, 成, 功, 着, 陆, 火]
目标: [问, 一, 号, 成, 功, 着, 陆, 火, 星]
      ↑ 每个位置预测下一个字
```

这就是语言模型的本质——**next token prediction（下一个词预测）**。GPT、Qwen、Claude 全都是这么训的。

---

## 二、模型内部：一步步走一遍

以输入 **"天问一号"** 为例，完整走一遍模型内部的计算。

### 2.1 Token Embedding —— 把字变成向量

Embedding 就是一张查找表（`nn.Embedding`），每个字对应一行向量：

```
"天" → id=12 → 查表 → [0.12, -0.34, 0.56, ..., 0.08]  ← 64个数字
"问" → id=49 → 查表 → [0.45, 0.22, -0.11, ..., 0.33]
"一" → id=2  → 查表 → [-0.09, 0.67, 0.44, ..., -0.21]
"号" → id=10 → 查表 → [0.33, -0.55, 0.12, ..., 0.77]
```

这些向量是**随机初始化**的，训练过程中会不断更新。训练结束后，语义相近的字会有相近的向量。

对应代码：
```python
self.token_embedding = nn.Embedding(vocab_size, d_model)  # (54, 64)
```

### 2.2 Positional Encoding —— 加位置信息

Transformer 没有循环结构（不像 RNN 一个字一个字顺序处理），它一次性看到所有字。问题来了：

```
"我爱你" 和 "你爱我" → Embedding 加起来完全一样！
```

所以要给每个位置加一个独特的 **"位置编码"**：

```
位置0的编码: [sin(0), cos(0), sin(0), cos(0), ...]
位置1的编码: [sin(1/10000^0), cos(1/10000^0), sin(1/10000^(2/64)), ...]
位置2的编码: [sin(2/10000^0), cos(2/10000^0), ...]
...
```

用 sin/cos 的好处：
- 每个位置的编码都不一样
- 不需要学习，是固定的数学公式
- 理论上可以处理任意长度的序列

```
最终输入 = Embedding向量 + 位置编码向量
```

### 2.3 Self-Attention —— Transformer 的灵魂

**核心问题：序列中的每个字，应该"关注"哪些其他字？**

每个字通过三个线性变换，生成三个向量：

```
Q (Query)  —— "我在找什么？"        通过 W_q 矩阵得到
K (Key)    —— "我有什么特征？"      通过 W_k 矩阵得到
V (Value)  —— "我的实际内容"        通过 W_v 矩阵得到
```

然后用 Q 和所有 K 做**点积**，得到注意力分数。以 **"号"** 这个字为例：

```
第一步：计算注意力分数（Q·K 点积）

  Q_号 · K_天 = 3.2   ← 高分，很相关
  Q_号 · K_问 = 0.1   ← 低分，不太相关
  Q_号 · K_一 = 0.5
  Q_号 · K_号 = 1.8

第二步：除以 √d（缩放，防止数值太大导致 softmax 梯度消失）

  [3.2, 0.1, 0.5, 1.8] / √64 = [0.40, 0.01, 0.06, 0.23]

第三步：softmax 归一化（变成概率分布，和为1）

  → [0.88, 0.00, 0.02, 0.10]

第四步：用权重加权求和各字的 V

  输出 = 0.88 × V_天 + 0.00 × V_问 + 0.02 × V_一 + 0.10 × V_号
```

**结论："号"最关注"天"（权重 0.88）**——因为"天问一号"是一个整体，模型学会了"号"要回头看"天"来理解语义。

数学公式：

```
Attention(Q, K, V) = softmax(Q · K^T / √d_k) · V
```

对应代码：
```python
scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_model)
attention_weights = F.softmax(scores, dim=-1)
output = torch.matmul(attention_weights, V)
```

### 2.4 Causal Mask —— 不能偷看未来

语言模型是一个字一个字往后生成的，训练时也不能让它偷看后面的字：

```
处理 "天" 时: 只能看 [天]             → 预测 "问"
处理 "问" 时: 只能看 [天, 问]          → 预测 "一"
处理 "一" 时: 只能看 [天, 问, 一]      → 预测 "号"
处理 "号" 时: 只能看 [天, 问, 一, 号]  → 预测 "成"
```

实现方式：在 attention score 上盖一个**下三角 mask**，把右上角设为 -∞：

```
         天    问    一    号
  天  [ 0.4   -∞    -∞    -∞  ]
  问  [ 0.1   0.3   -∞    -∞  ]     softmax 后 -∞ → 0
  一  [ 0.2   0.1   0.5   -∞  ]     右上角注意力权重全为 0
  号  [ 0.4   0.01  0.06  0.23]
```

这就是注意力可视化中右上角全是 0.00 的原因。

对应代码：
```python
mask = torch.tril(torch.ones(seq_len, seq_len))  # 下三角矩阵
scores = scores.masked_fill(mask == 0, float('-inf'))
```

### 2.5 Multi-Head Attention —— 多角度关注

一个 Attention 头只能学一种关注模式。我们用 4 个头，各学各的：

```
d_model=64, n_heads=4 → 每个头处理 64/4=16 维

Head 1 (16维): 可能学会关注"前一个字"
Head 2 (16维): 可能学会关注"语义相关的字"
Head 3 (16维): 可能学会关注"句子开头"
Head 4 (16维): 可能学会关注"标点位置"
```

4 个头的输出拼起来（16×4=64），再过一个线性层融合：

```python
# 拆成多头
Q = Q.view(batch, seq_len, n_heads, head_dim).transpose(1, 2)
# → (batch, 4, seq_len, 16)

# 各自做 attention...

# 拼回来
output = output.transpose(1, 2).view(batch, seq_len, d_model)
output = self.W_o(output)  # 线性融合
```

### 2.6 Feed-Forward Network —— 消化信息

Attention 负责"看其他位置"，FFN 负责"消化这些信息"：

```
输入(64维) → 升维(64→256) → ReLU激活 → 降维(256→64) → 输出(64维)
```

为什么要先升维再降维？升维到更高的空间做非线性变换，可以学到更复杂的特征。

```python
self.fc1 = nn.Linear(d_model, 4 * d_model)   # 64 → 256
self.fc2 = nn.Linear(4 * d_model, d_model)    # 256 → 64
# forward: fc2(ReLU(fc1(x)))
```

### 2.7 残差连接 + LayerNorm

一个 Transformer Block 的完整结构：

```
输入 x
  │
  ├──────────────────┐
  ↓                  │
  LayerNorm          │
  ↓                  │
  Multi-Head Attn    │
  ↓                  │
  + ←────────────────┘  残差连接: output = x + Attention(LayerNorm(x))
  │
  ├──────────────────┐
  ↓                  │
  LayerNorm          │
  ↓                  │
  FFN                │
  ↓                  │
  + ←────────────────┘  残差连接: output = x + FFN(LayerNorm(x))
  │
  ↓
输出
```

**残差连接**的作用：让梯度能直接流过去。没有残差的话，4 层还好，24 层（Qwen2.5）或 120 层（GPT-4）就训不动了。

**LayerNorm**：对每个位置的向量做归一化（均值→0，方差→1），让训练更稳定。

### 2.8 输出层 —— 从向量到概率

经过 4 个 Block 后，每个位置得到一个 64 维向量。最后一个线性层把它映射到词表大小（54）：

```
64维向量 → Linear(64, 54) → 54个分数(logits)
                            ↓ softmax
                         54个概率

位置"号"的输出:
  P("成")=0.82  ← 最高！预测下一个字是"成" ✓
  P("天")=0.03
  P("功")=0.01
  ...其余51个字的概率都很小
```

## 三、训练过程

### 3.1 Loss 的含义

Loss 用的是**交叉熵**（Cross Entropy），就是"模型预测概率"的负对数：

```
如果模型对正确答案的预测概率是 P，则 Loss = -log(P)

P = 1.0 (完全确定) → Loss = 0      ← 完美
P = 0.5 (半猜半不猜) → Loss = 0.69
P = 0.02 (瞎猜)     → Loss = 3.91  ← 很差
```

### 3.2 训练曲线解读

```
Epoch  10  Loss=3.43 → 54个字随机猜的理论值是 -ln(1/54)=3.99
                       模型比纯随机好一点点，刚开始学

Epoch  30  Loss=1.89 → e^(-1.89)=15%，对正确答案的平均置信度 15%
                       开始记住一些高频模式了

Epoch  50  Loss=0.82 → e^(-0.82)=44%，快一半了
                       大部分常见搭配已经学会

Epoch 100  Loss=0.10 → e^(-0.10)=90%，非常确定
                       基本记住了所有训练语料
```

### 3.3 反向传播

每轮训练做三件事：

```
1. 前向传播：输入 → 模型 → 预测 → 和正确答案比 → 得到 Loss
2. 反向传播：Loss → 对每个参数计算梯度（该调大还是调小、调多少）
3. 更新参数：参数 = 参数 - 学习率 × 梯度
```

对应代码：
```python
logits, _ = model(inputs)                                  # 前向
loss = F.cross_entropy(logits.view(-1, vocab_size), targets.view(-1))  # 算Loss
loss.backward()                                            # 反向
optimizer.step()                                           # 更新
```

## 四、生成过程

生成就是不断重复 "预测下一个字"：

```
第1步: 输入 "天问"
       → 模型对"问"这个位置的输出: P("一")=0.85 最高
       → 选 "一"

第2步: 输入 "天问一"
       → 模型对"一"这个位置的输出: P("号")=0.91 最高
       → 选 "号"

第3步: 输入 "天问一号"
       → P("成")=0.82 最高
       → 选 "成"

...一直续写到指定长度
```

这就是 **自回归生成（Autoregressive Generation）**——每一步都把上一步的输出拼到输入后面，再预测下一个。**所有大模型生成文本的方式都是这样**。

代码中用了 temperature=0.8 的采样，不是每次都选概率最高的，而是按概率随机采样，所以每次运行结果可能略有不同。

## 五、注意力可视化怎么看

demo 输出了 "天问一号成功着陆火星" 的注意力权重矩阵：

```
           天     问     一     号     成     功     着     陆     火     星
  天  │ 1.00  0.00  ...                                    ← 第一个字只能看自己
  问  │ 0.15  0.85  ...                                    ← 主要看自己
  号  │ 0.88  0.00  0.02  0.10  ...                        ← 最关注"天"
  火  │ 0.01  0.02  ...  0.93  0.00  ...                   ← 最关注"陆"
  星  │ 0.16  0.01  ...  0.75  0.00  0.01                  ← 最关注"陆"
```

**怎么看：**
- 每一行是一个字，表示它在"关注"谁
- 数值越大，表示关注度越高
- 右上角全是 0 → causal mask 的效果，不能看未来
- 第一个字（"天"）只能看自己，所以权重 = 1.00

**有趣的发现：**
- "号" 最关注 "天"（0.88）：说明模型学会了"天问一号"是一个整体
- "火" 最关注 "陆"（0.93）：着**陆**→**火**星，"陆"提示了接下来是地点
- "星" 最关注 "陆"（0.75）：同上，"陆"对预测"火星"很重要

注意：这只是第 1 层、第 1 个头的注意力。4 层 × 4 头 = 16 个注意力矩阵，每个关注不同的模式。

## 六、和真实大模型的对比

| | 我们的 demo | Qwen2.5-0.5B | GPT-4 |
|------|-----------|-------------|-------|
| 参数量 | 0.21M | 500M | ~1,800,000M |
| 层数 | 4 | 24 | ~120 |
| 注意力头数 | 4 | 14 | ~96 |
| 隐藏维度 (d_model) | 64 | 896 | ~12,288 |
| FFN 维度 | 256 | 4,864 | ~49,152 |
| 词表大小 | 54（字符） | 151,936（子词） | ~100,000 |
| 训练数据 | 133 字 | 几万亿 token | 更多 |
| 训练时间 | 几秒 | 几周 | 几个月 |

**结构完全一样，只是规模不同。**

从 demo 到 GPT-4，本质上就是三件事：
1. 把模型**堆大**（更多层、更宽、更多头）
2. 把数据**喂多**（从 133 字到几万亿 token）
3. 把训练**拉长**（从几秒到几个月）

理解了这个 demo，你就理解了所有大模型的骨架。
